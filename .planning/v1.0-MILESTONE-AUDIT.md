## Critical Gaps (P0)

### Gap 1: Production Scheduler Never Polls SNMP

**Severity:** P0 — the system runs but never monitors the printer

**What happens:** Every hour, `run_job()` in `main.py` calls `graph.invoke(_build_initial_state())`. `_build_initial_state()` sets `poll_result=None`. The `analyst` node receives `None` and immediately returns `alert_needed=False` with a log message. No SNMP poll is performed. No threshold check runs. No email is ever sent.

**Root cause:** `main.py` bypasses `run_pipeline()` (which correctly calls `SNMPAdapter.poll()` in `supervisor.py` lines 136–140) and invokes the LangGraph graph directly with an empty initial state.

**Fix required in:** `main.py` `run_job()` (lines 167–186) — must call `SNMPAdapter(host, community).poll()` before `graph.invoke()` and inject the result into the initial state.

**Affected requirements:** SNMP-01, ANLZ-01, GURD-01, GURD-02, GURD-03, GURD-04, ALRT-01, ALRT-02, ALRT-03, SCHD-01

---

### Gap 2: Poll Results Never Written to JSONL

**Severity:** P0 — SNMP-04 unsatisfied, LLM trend analysis permanently in cold-start

**What happens:** `append_poll_result()` is called only for error conditions (`llm_failure`, `suppressed_alert`, `pipeline_error`). Normal toner `PollResult` data from SNMP is never written to `logs/printer_history.jsonl`. `read_poll_history()` always returns empty list. `compute_color_stats()` always gets `n=0` readings. The LLM analyst is permanently in cold-start mode (n < 3 guard), never producing trend-aware confidence scores or urgency upgrades.

**Fix required in:** `main.py` `run_job()` (or `run_analyst()` at the top of the function) — must call `append_poll_result(poll_result)` after a successful SNMP poll so history accumulates.

**Affected requirements:** SNMP-04, ANLZ-04 (trend unreachable), ANLZ-02/ANLZ-03 (LLM only reachable after n≥3 polls)

## Partial Implementation Gaps (P1)

### Gap 3: LLM Confidence Score Absent as Standalone Email Field

**Severity:** P1 — ALRT-02 partially satisfied

**What happens:** `build_body()` in `agents/communicator.py` includes `"Analysis:"` + `llm_reasoning` text but does not add an explicit `"Confidence: X%"` line. ALRT-02 specifies confidence score as a distinct field in the alert email.

**Fix required in:** `agents/communicator.py` `build_body()` (lines 75–128) — add `f"Confidence: {llm_confidence:.0%}"` (or similar) as a distinct line when `llm_reasoning` is present.

## Summary

Project Sentinel v1.0 has 103 passing tests and all 4 phases verified individually. The LangGraph pipeline, policy guard, SMTP alerting, LLM analysis, and APScheduler are all implemented and tested in isolation.

**However**, integration checking revealed that the production entry point (`main.py` `run_job()`) never calls `SNMPAdapter.poll()`. Every scheduled cycle starts with `poll_result=None` and the analyst immediately returns without monitoring the printer. This is a wiring gap between Phase 4's `build_graph()` architecture and `run_pipeline()`'s SNMP invocation logic.

These 2 P0 gaps require a gap-closure phase before v1.0 can be considered production-ready.
