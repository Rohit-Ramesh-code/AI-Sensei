    **compute_color_stats(color, log_path, window_days=7) -> dict:**
    - Load history via read_poll_history(log_path=log_path)
    - Filter: skip records where record.get("event_type") is not None (these are suppression/failure events)
    - Filter: only include records where timestamp >= now - timedelta(days=window_days)
    - For each qualifying record, extract readings where reading["color"] == color and reading.get("toner_pct") is not None
    - Collect (timestamp_str, toner_pct) pairs to compute actual velocity
    - n = total readings collected
    - If n < 2: return {"n": n, "std_dev": None, "velocity_pct_per_day": None}
    - Compute std_dev = statistics.stdev([pct for _, pct in pairs])
    - Compute velocity using real timestamps: first and last entry's timestamps
      ```python
      first_ts = datetime.fromisoformat(pairs[0][0])
      last_ts = datetime.fromisoformat(pairs[-1][0])
      elapsed_hours = max((last_ts - first_ts).total_seconds() / 3600, 1.0)  # avoid division by zero
      velocity_pct_per_day = (pairs[-1][1] - pairs[0][1]) / elapsed_hours * 24
      ```
    - Return: {"n": n, "std_dev": round(std_dev, 2), "velocity_pct_per_day": round(velocity_pct_per_day, 3)}
    - Guard: if n < 2 after filtering, velocity and std_dev are None

    **run_analyst(state) -> AgentState — updated logic:**

    The function now has three paths per flagged color:
    1. **Cold start** (stats["n"] < 3): skip LLM, use deterministic urgency, llm_confidence=None, llm_reasoning=None for that color
    2. **LLM success**: set llm_confidence = result.confidence, llm_reasoning = result.reasoning; override urgency using trend_label
    3. **LLM failure** (call_llm_analyst returns None): use deterministic urgency, llm_confidence=None, llm_reasoning=None

    The overall flow:
    ```
    1. Extract flagged_colors using existing deterministic threshold logic (keep current logic intact)
    2. If alert_needed is False: set llm_confidence=None, llm_reasoning=None, return
    3. For EACH flagged color:
       a. compute_color_stats(color, log_path)
       b. if stats["n"] < 3: log cold start, skip LLM for this color
       c. else: call call_llm_analyst(color, toner_pct, stats, log_path)
          - if result: set color's urgency from trend_label mapping, accumulate confidence and reasoning
          - if None: log LLM failure, use existing deterministic urgency
    4. Set state["llm_confidence"] = average/lowest confidence across LLM-analyzed colors (or None if no LLM was called)
    5. Set state["llm_reasoning"] = combined reasoning string (or None if no LLM was called)
    ```

    For urgency mapping from trend_label:
    - "Critically low" → "CRITICAL"
    - "Declining rapidly" → "CRITICAL"
    - "Declining slowly" → "WARNING"
    - "Stable" → keep deterministic urgency (anomalous — LLM says stable but threshold triggered)

    For combining confidence across multiple colors:
    - Use the minimum confidence across all LLM-analyzed colors (conservative: alert gates on weakest confidence)
    - If no colors were LLM-analyzed (all cold start or all LLM failure): llm_confidence = None

    For combining reasoning across multiple colors:
    - Concatenate per-color reasoning with "\n" separator if multiple colors
    - If only one color: use that color's reasoning directly

    Also add log_path parameter to run_analyst() signature (with default from persistence.LOG_PATH) for test isolation:
