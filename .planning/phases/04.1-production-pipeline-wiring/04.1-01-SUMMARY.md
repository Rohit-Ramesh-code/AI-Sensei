## Decisions Made

- SNMPAdapter imported at main.py module scope (same group as append_poll_result) — allows patching via `main.SNMPAdapter` in tests without module reload
- run_job() ordering: snmp.poll() -> append_poll_result(poll_result) -> graph.invoke() — persistence before graph ensures history accumulates even if graph raises
- Confidence: shown only when llm_reasoning is not None — confidence score without analysis context is meaningless to the recipient
- f"{llm_confidence:.0%}" format: 0.91 -> "91%", 0.85 -> "85%" — clean human-readable percentage
- Test helper _run_job_under_patches() calls job_fn() INSIDE the with patch() context block — the run_job closure references main.SNMPAdapter, which only resolves to the mock when the patch is active
- TestRunJobErrorBoundary updated: assert_called_once() -> check call_args_list for pipeline_error entry — append now called twice on error path (once for poll_result, once for error event)

- **Found during:** Task 1 (TDD RED -> GREEN)
- **Issue:** The initial test helper called job_fn() outside the with patch() block. The run_job closure captures main.SNMPAdapter by name, so calling it after patches are removed caused the real SNMPAdapter (and real SNMP network call) to be invoked
- **Fix:** Redesigned _run_job_under_patches() helper to call job_fn() inside the with patch() context; changed from instance method to module-level function for clarity
- **Files modified:** tests/test_main.py
