<objective>
Close the three P0/P1 production wiring gaps found in the v1.0 milestone audit.

Gap 1 (P0, SNMP-01, SCHD-01): run_job() in main.py calls graph.invoke(_build_initial_state()) where poll_result=None. The analyst exits immediately. SNMP is never polled in production.

Gap 2 (P0, SNMP-04, ANLZ-04): append_poll_result() is never called for normal toner PollResult data. The JSONL history is always empty. compute_color_stats() is permanently in cold-start (n < 3). Trend-aware urgency (ANLZ-04) is structurally unreachable.

Gap 3 (P1, ALRT-02): build_body() in agents/communicator.py has no llm_confidence parameter. The alert email includes LLM reasoning text but never an explicit "Confidence: X%" line. ALRT-02 requires confidence as a distinct structured field.

Purpose: Make the system actually monitor the printer (Gaps 1+2) and satisfy the full ALRT-02 email content requirement (Gap 3). No new libraries. All gaps are missing function calls and missing parameters — not missing infrastructure.

def build_body(
    printer_host: str,
    flagged_colors: list[dict],
    llm_reasoning: "str | None" = None,
    llm_confidence: "float | None" = None,   # NEW — add after llm_reasoning
) -> str:
    lines = [f"Printer: {printer_host}", "", "Low toner detected:"]

    for entry in flagged_colors:
        color = entry["color"]
        display_value = entry["display_value"]
        urgency = entry["urgency"]
        lines.append(f"  {color.capitalize()}: {display_value} [{urgency}]")
        lines.append(f"  Recommended action: Order {color} toner")
        lines.append("")

    if llm_reasoning is not None:
        if llm_confidence is not None:
            lines.append(f"Confidence: {llm_confidence:.0%}")  # e.g. "Confidence: 91%"
        lines.append("Analysis:")
        lines.append(llm_reasoning)
    else:
        lines.append("Note: LLM analysis unavailable — alert based on threshold check only.")

    return "\n".join(lines)
