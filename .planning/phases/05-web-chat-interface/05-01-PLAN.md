```python
    GET  /      — Serves templates/chat.html

    TONER_ALERT_THRESHOLD    — Warning threshold % (default: 20)
    TONER_CRITICAL_THRESHOLD — Critical threshold % (default: 10)

Run:
    python chat_server.py
"""
```

Module-level constants (outside create_app — safe as they have no side effects):
- VALID_ACTIONS: frozenset of the 4 valid action strings
- SYSTEM_PROMPT: the exact intent classification prompt from RESEARCH.md
- SUPPRESSION_MESSAGES dict with prefix-based matching logic (see _plain_english below)

Helper functions (module-level, pure functions, no side effects):
- `_envelope(status: str, action: str, data: dict) -> dict` — returns the standard JSON envelope with UTC ISO timestamp
- `_plain_english(reason: str | None) -> str | None` — maps suppression reason to plain English using substring matching (not exact match, because suppression_reason strings are dynamic — e.g. "rate_limit: last_alert=..." should match "rate_limit:"). Mapping:
  - reason starts with "rate_limit:" → "An alert was already sent in the last 24 hours."
  - reason starts with "data_quality:" → "SNMP data quality check failed (stale, null, or invalid readings)."
  - "reason=erratic_readings" in reason → "Toner readings were inconsistent — alert withheld to avoid a false alarm."
  - "reason=low_confidence" in reason → "The LLM's confidence score was too low to trigger an alert reliably."
  - fallback → return reason as-is (do not crash on unknown future reasons)
- `classify_intent(message: str) -> str` — builds Client(host=os.getenv("OLLAMA_BASE_URL", "http://localhost:11434")), calls client.chat() with SYSTEM_PROMPT and format="json", options={"temperature": 0}. Parses response.message.content as JSON, extracts "action" key, validates against VALID_ACTIONS, returns "unknown" on any exception or invalid action. DO NOT rely on OLLAMA_HOST env var — always use explicit host from OLLAMA_BASE_URL.
- `_toner_dict_from_poll(poll: dict) -> dict` — converts PollResult readings list to {color: {pct, status}} dict. Status: pct is None → use quality_flag value as status; pct <= TONER_CRITICAL_THRESHOLD → "critical"; pct <= TONER_ALERT_THRESHOLD → "low"; else → "ok". Read thresholds from TONER_ALERT_THRESHOLD and TONER_CRITICAL_THRESHOLD env vars inside the function (not at module level).

`create_app() -> Flask`:
- Call load_dotenv() as the FIRST LINE inside create_app() — before any project imports. This matches the pattern established in Phase 4 (main.py). IMPORTANT: load_dotenv() inside factory, NOT at module level — prevents test isolation breakage.
- Import project modules (snmp_adapter, persistence, supervisor) AFTER load_dotenv() inside create_app(), not at module top level. This prevents import-time side effects from running before test fixtures patch env vars.
- Register GET / route: return render_template("chat.html")
- Register POST /chat route:
  - Parse body with request.get_json(force=True, silent=True) or {}
  - Extract message = str(body.get("message", "")).strip()

  - Return jsonify(result) with 200
- Return app
