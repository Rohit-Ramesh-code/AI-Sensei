Implement adapters/snmp_adapter.py. This file has two concerns: (1) the sentinel classification logic and (2) the SNMPAdapter class that either polls the real printer or returns fixture data.

Structure:

1. classify_snmp_value(raw_level: int | None, max_capacity: int) -> tuple[QualityFlag, float | None]
   Classification rules (apply in order):
   - raw_level is None → QualityFlag.NULL_VALUE, None
   - raw_level == -2   → QualityFlag.UNKNOWN, None
   - raw_level == -3   → QualityFlag.BELOW_LOW_THRESHOLD, None
   - raw_level == -1   → QualityFlag.NOT_SUPPORTED, None
   - max_capacity <= 0 → QualityFlag.UNKNOWN, None  (max_capacity itself is a sentinel)
   - not (0 <= raw_level <= max_capacity) → QualityFlag.OUT_OF_RANGE, None
   - else              → QualityFlag.OK, round((raw_level / max_capacity) * 100.0, 1)

2. Mock fixture (MOCK_FIXTURE dict) representing a typical Lexmark XC2235 state:
   - black:   level=85, max=100       → OK, 85.0%
   - cyan:    level=42, max=100       → OK, 42.0%
   - magenta: level=-2, max=100       → UNKNOWN (sentinel)
   - yellow:  level=-3, max=100       → BELOW_LOW_THRESHOLD (sentinel)

3. SNMPAdapter class:
   - __init__(self, host: str, community: str, use_mock: bool = False)
     Read use_mock from parameter; also check os.getenv("USE_MOCK_SNMP", "false").lower() == "true"
   - poll(self) -> PollResult
     If mock: build PollResult from MOCK_FIXTURE using classify_snmp_value for each color.
     If real: use pysnmp-sync-adapter to:
       a. Walk prtMarkerSuppliesDescription (OID 1.3.6.1.2.1.43.11.1.1.6.1.{1-8}) to build dynamic color→index map. Never hardcode index order — the XC2235 may use any ordering.
       b. For each color in the map, GET prtMarkerSuppliesLevel (1.3.6.1.2.1.43.11.1.1.9.1.{idx}) and prtMarkerSuppliesMaxCapacity (1.3.6.1.2.1.43.11.1.1.8.1.{idx}).
       c. Call classify_snmp_value for each (level, max_capacity) pair.
       d. Return PollResult with all four readings and overall_quality_ok computed.
     If SNMP transport fails: catch the exception, return a PollResult with snmp_error set to str(e), all readings having QualityFlag.SNMP_ERROR, and overall_quality_ok=False. Do NOT raise — the adapter must always return a PollResult, never crash the pipeline.

4. Import from state_types: QualityFlag, TonerReading, PollResult
   Use: from pysnmp_sync_adapter import get_cmd_sync
   Use: from pysnmp.hlapi.v3arch.asyncio import CommunityData, UdpTransportTarget, ContextData, ObjectType, ObjectIdentity, SnmpEngine
   Timestamp: datetime.now(timezone.utc).isoformat()

CRITICAL anti-patterns (per research):
- NEVER use next(getCmd(...)) — pysnmp v7 is asyncio-only; that pattern returns a coroutine, not a value
- NEVER hardcode color index order — walk prtMarkerSuppliesDescription first
- NEVER pass raw sentinel integers (-2, -3) in any TonerReading field — always convert via classify_snmp_value
- NEVER open the log file in write mode — that is handled in Plan 02, not here
